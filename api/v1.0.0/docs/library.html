The Official JavaScript library for CodeCBT

This library takes away the stress of you fetching the data from our servers. it helps you to make these calls using pre-created made methods, all you need do is to setup the JS file and make the call. (This library is what the CodeCBT demo project sits on View demo)

Download code_cbt.js v1.0.0 (1.32kb)

Setting up the environment

There are two ways to get the library up and running. The first is by using the code_cbt.js CDN, and the second by downloading the file.

code_cbt.js CDN

CDN link:
http://127.0.0.1:5500/code_cbt_lib/v1.0.0/code_cbt.js

Downloading the code_cbt.js file

The code cbt library is quite small. The archieved file is only 1.32Kb and the JS file when uncompressed is about 5.1Kb. Click here to begin download (v1.0.0);

Getting Started

With the files or CDN connections established, put the code_cbt library above other script tags (if you are working with the browser) so you can be able to use the functions.

Code CBT.js Methods

The CodeCBT constructor makes use of static methods so these methods can be called by inserting the method name after the constructor name (eg: CodeCBT.< method name >). Below all Methods that help make the fetch calls for you are shown with detailed examples and explanation.

  • CodeCBT .js makes calls to specific categories and files we specify. A list of all available categories and category initials can be found here.
  • All methods return a promise and they do a good job in logging errors to the console.

CodeCBT.getData

This method makes http fetch calls to the specified category and file we give to it. It returns all the data it finds in the specified json file.

Getting all data in a specified file
getData(< category >, < fileName >)
Response (if < category > is javascript and < fileName > = js.json)
{
"category": "JavaScript",
"category_initial": "js",
"questions": [
{
"id": "js1",
"source": "code CBT",
"question": "What does XHR stand for",
"answer": "XML Http Request",
"options": [
"Xtensive Http Response",
"Xtensive Http Request",
"XML Http Request",
"XML Http Response"
]
}...
]
}

CodeCBT.getQuestionByIndex

This method takes three arguements. The first is the category, followed by the file name and lastly (but not the least), the questionIndex. It returns the question at the given index (not the question id).

Getting a question by its index
getQuestionByIndex(< category >, < fileName >, < questionIndex >)
Response (if < category > is javascript, < fileName > = js.json and < questionIndex > = 3 )
{
{
"id": "js4",
"source": "code CBT",
"question": "All of the following are ways to handle asyncronous calls in JavaScript except using",
"answer": "Reusables",
"options": [
"Callback functions",
"Async await",
"Reusables",
"Promises"
]
}
}

CodeCBT.getQuestionById

This method has same syntax as the getQuestionByIndex method but it returns the question whose id corresponds with the specified id (the question id is simply its index + 1).

CodeCBT.getRandomQuestions

This will return an array containing a number of random questions according to the number of questions we pass it to return (the number of questions to return must be a less than the number of questions in the database -1, eg: if the number of questions in the web.json file of the web_dev category is 51 then the highest no of questions we can request is 50).

CodeCBT.getRandomQuestion

This is simmilar to the getRandomQuestions method but it returns a single random Question. Every other feature is Exactly the samewith the getRandomQuestions method.

Supported Browsers

The code_cbt javaScript library supports all modern browsers but it does not support Internet Explorer 11 downwards. Practically any browser which does not support ES6 Classes is not supported. WE ARE WORKING ON THIS ISSUE, THANKS FOR UNDERSTANDING.
